# 社群內容審核系統 SPEC（重製範例）

> **本文件為作品展示用之重製範例。** 以虛構的影音社交產品「Lark」為題材，展示本人撰寫後台功能規格的方法與顆粒度。所有數值、規則與介面均為虛構，不含任何前雇主機密資訊。

| 文件資訊 | |
|---|---|
| 模組 | 後台 > 社群管理（貼文審核／自動審核名單／屏蔽字詞／檢舉處理） |
| 讀者 | 前端、後端、QA、營運 |
| 配套原型 | Axure 原型另附（本範例以文字重現關鍵畫面結構） |

## 名詞定義

| 名詞 | 定義 |
|---|---|
| 貼文 | 用戶於前台發布之圖文或影音內容，含正文說明文字。 |
| 審核狀態 | 貼文的生命週期狀態：`待審核`、`已通過`、`已拒絕`。 |
| 自動審核名單 | 決定用戶發文是否免人工審核的帳號分級：`一般會員`、`白名單`、`黑名單`。 |
| 前台顯示 | 營運對已通過貼文的二次控制：`正常`（可被推薦）、`隱藏`（僅發布者可見）。 |

## 1. 審核管線總覽

用戶發布貼文後，依發布者的「自動審核名單」分級進入不同流程：

```
用戶發布貼文
   │
   ├─ 發布者為【白名單】 → 免人工審核，直接「已通過」，可進推薦池
   │                        （營運仍可事後於貼文審核頁改為「已拒絕」）
   │
   ├─ 發布者為【黑名單】 → 直接「已拒絕」，僅發布者個人頁自己可見「審核中」
   │                        （營運仍可事後改為「已通過」）
   │
   └─ 發布者為【一般會員】 → 進入人工審核佇列（狀態：待審核）
                              │
                              ├─ 營運「通過」 → 已通過，進推薦池
                              └─ 營運「拒絕」 → 已拒絕
```

設計要點：

1. **白名單不是免死金牌**：白名單貼文仍寫入審核列表（狀態預設已通過），營運可回溯改判，避免白名單帳號被盜後成為破口。
2. **黑名單靜默處理**：黑名單用戶發文時前台不提示「被拒絕」，發布者視角永遠顯示「審核中」，降低對抗行為（換帳號重發、規避偵測）。
3. **審核與露出分離**：「審核狀態」決定內容合不合規，「前台顯示」讓營運在不推翻審核結論的前提下控制推薦露出，兩軌互不覆寫。

## 2. 貼文審核（後台 > 社群管理 > 貼文審核）

### 2.1 頁面結構

搜尋列（發布時間區間／會員帳號／審核狀態）＋快捷操作列（一鍵通過／一鍵拒絕）＋申請列表（依發布時間降冪）。

### 2.2 欄位說明

| 項目 | 說明（預設不顯示任何資料，需點擊搜尋） |
|---|---|
| 發布時間 | 供選擇發布之時間區間，預設近 7 日。 |
| 會員帳號 | 供輸入發布者帳號，精準匹配。 |
| 審核狀態 | 全部／待審核／已通過／已拒絕。預設「待審核」。 |
| 勾選框 | 供多選申請；點擊表頭勾選當頁全選。 |
| 發布內容 | 顯示貼文縮圖，點擊開啟彈窗顯示完整內容；影音自動開始播放；多媒體時可於彈窗內左右切換。 |
| 貼文說明 | 顯示正文文字；逾 500 字截斷並顯示「詳目」，點擊開啟彈窗顯示完整內容。 |
| 觀看權限 | 顯示發布者設定的可見範圍：所有人／互相關注／粉絲／僅自己。 |
| 審核狀態 | 待審核／已通過／已拒絕（定義見影響矩陣）。 |
| 前台顯示 | 正常／隱藏（僅「已通過」之貼文可切換）。 |
| 操作：通過 | 僅於「待審核」與「已拒絕」時顯示。點擊彈出二次確認氣泡：確認→狀態切換為「已通過」；取消→無行為。 |
| 操作：拒絕 | 僅於「待審核」與「已通過」時顯示。點擊彈出二次確認氣泡：確認→狀態切換為「已拒絕」；取消→無行為。 |

### 2.3 審核狀態 × 前台露出影響矩陣

狀態多的功能不用散文描述，用矩陣把每一個組合釘死：

| 露出面 | 待審核 | 已拒絕 | 已通過＋正常 | 已通過＋隱藏 |
|---|---|---|---|---|
| 首頁推薦池 | 不出現 | 不出現 | 可出現 | 不出現 |
| 個人頁（發布者自己看） | 顯示，標「審核中」 | 顯示，標「審核中」* | 顯示 | 顯示 |
| 個人頁（其他人看） | 不顯示 | 不顯示 | 顯示 | 不顯示 |
| 收藏夾（曾收藏者） | 不顯示 | 不顯示 | 顯示 | 不顯示 |
| 站外分享連結 | 開啟顯示「內容審核中，請稍後再試」 | 開啟顯示「內容已下架或涉違規」 | 正常開啟 | 開啟顯示「內容已下架或涉違規」 |

\* 黑名單靜默策略：發布者自己視角永遠是「審核中」，不揭露已被拒絕。

## 3. 自動審核名單（後台 > 社群管理 > 自動審核）

### 3.1 欄位說明

| 項目 | 說明 |
|---|---|
| 搜尋欄位 | 會員帳號＋名單類型（全部／一般會員／白名單／黑名單）。 |
| 快捷選項 | 一鍵一般會員／一鍵白名單／一鍵黑名單：對勾選會員批次切換分級，均需二次確認。 |
| 會員列表 | 以最近一次分級變更時間降冪排序。 |
| 自動審核類型 | 一般會員：發文走人工審核。白名單：發文自動通過（可回溯改判）。黑名單：發文自動拒絕（可回溯改判）。 |
| 操作 | 依當前分級顯示另外兩種分級之切換按鈕，均需二次確認。 |

### 3.2 設計要點

- 三種分級互斥，切換即時生效，**不回溯**已產生的審核結果（歷史貼文狀態不變）。
- 分級變更寫入操作日誌（操作者、時間、原分級、新分級），供爭議追查。

## 4. 屏蔽字詞（後台 > 社群管理 > 屏蔽字詞）

### 4.1 欄位與新增規則

| 項目 | 說明 |
|---|---|
| 搜尋欄位 | 供輸入屏蔽字詞搜尋（模糊匹配）。 |
| 字詞列表 | 屏蔽字詞＋設定時間，依設定時間降冪。 |
| 新增 | 彈窗輸入字詞→確定儲存。送出時一律轉成小寫；重複時擋下並提示「該屏蔽字詞已存在」。 |
| 刪除 | 二次確認後刪除，重整列表。 |

### 4.2 生效範圍與上限

| 規則 | 定義 |
|---|---|
| 命中判定 | 不分大小寫；比對時去除字詞間空白（防「屏 蔽 字」規避）。 |
| 留言 | 命中即整條留言僅發布者自己可見，其他人（含創作者）皆看不到；**不區分留言早於或晚於字詞設定**，設定後即時全量生效。 |
| 個人簡介 | 儲存時擋下，提示「請勿包含 {命中字詞}」。 |
| 單一字詞長度上限 | 24 字。 |
| 字詞總數上限 | 150 組（達上限時「新增」按鈕置灰並提示）。 |

## 5. 檢舉處理（後台 > 社群管理 > 檢舉內容）

### 5.1 欄位說明

| 項目 | 說明 |
|---|---|
| 搜尋欄位 | 最新檢舉時間區間／檢舉帳號／被檢舉帳號／檢舉狀態（全部／待處理／檢舉駁回／檢舉通過）。 |
| 檢舉列表 | 同一則內容之多筆檢舉**合併為一列**，顯示：初次／最新檢舉時間、第一檢舉帳號、檢舉人數（次數）、被檢舉帳號、內容縮圖、檢舉狀態、前台顯示。 |
| 檢舉人數(次數) | 格式「去重人數(總次數)」，點擊開啟檢舉詳情彈窗。 |
| 檢舉詳情彈窗 | 逐筆列出：檢舉時間、行為（檢舉／通過／駁回）、操作帳號、檢舉理由。營運處理紀錄與用戶檢舉同列呈現，形成完整時間軸。 |
| 操作：通過／駁回 | 均需二次確認。通過＝內容違規，於前台隱藏；駁回＝內容正常，維持原狀。 |

### 5.2 檢舉狀態 × 審核狀態交叉關係表

檢舉與審核是兩條獨立軌道，交叉組合用表格釘死，任何一格都不能靠猜：

| 審核狀態＼檢舉狀態 | 未檢舉 | 待處理 | 檢舉駁回 | 檢舉通過 |
|---|---|---|---|---|
| 待審核 | 不進推薦池，僅發布者自己可見 | 同左 | 同左 | 同左（檢舉結論待審核通過後才有實質效果） |
| 已通過 | 可進推薦池 | 可進推薦池（不因被檢舉先行下架） | 可進推薦池 | 不進推薦池，僅發布者自己可見 |
| 已拒絕 | 僅發布者自己可見 | 同左 | 同左 | 同左 |

設計要點：**檢舉不即時下架**。待處理期間內容維持原露出，避免檢舉功能被濫用為攻擊工具（惡意檢舉打競爭對手）；確認違規（檢舉通過）才隱藏。

### 5.3 留言檢舉影響表

| 屏蔽情形＼檢舉狀態 | 未檢舉 | 待處理 | 檢舉駁回 | 檢舉通過 |
|---|---|---|---|---|
| 無屏蔽字詞 | 正常顯示 | 正常顯示 | 正常顯示 | 僅留言者自己可見 |
| 含屏蔽字詞 | 僅留言者自己可見 | 僅留言者自己可見 | 僅留言者自己可見 | 僅留言者自己可見 |

## 6. 通用規則

### 6.1 列表分頁

所有後台列表共用：總條數／單頁條數（30、50、100 可選，預設 30）／上下頁鈕／頁碼（當前頁前後兩頁＋首末頁，中略以 … 呈現）／快捷前往（輸入頁數按 Enter 跳轉；輸入大於最末頁時前往最末頁）。

### 6.2 二次確認

所有不可逆或批次操作（狀態切換、刪除、一鍵批次）一律彈出確認氣泡：確認→執行並關閉；取消→無行為。

## 7. 邊界案例 Q&A

規格評審會議中釐清的邊界，記錄於此供 QA 展開測試案例：

1. **已收藏的貼文事後被拒絕，收藏夾如何顯示？**
   → 於收藏夾直接消失不顯示；重新通過後恢復顯示。
2. **已分享至站外的貼文事後被拒絕？**
   → 開啟連結顯示「內容已下架或涉違規」，不顯示任何內容殘影。
3. **貼文正在審核中，站外連結被點開？**
   → 顯示「內容審核中，請稍後再看」。
4. **發布者註銷帳號或被封禁，其待審貼文如何處理？**
   → 不再出現於貼文審核列表；既有檢舉單保留供追溯，但檢舉列表標記「帳號已註銷」且不可再操作。
5. **檢舉人註銷帳號，檢舉單是否保留？**
   → 保留且不影響處理，檢舉帳號欄顯示「已註銷用戶」；人數統計不回滾。
6. **同一用戶對同一內容重複檢舉？**
   → 次數累計、人數不重複計，檢舉詳情逐筆保留。
7. **自動審核分級變更是否影響歷史貼文？**
   → 不回溯。分級僅影響變更後的新發布內容。
