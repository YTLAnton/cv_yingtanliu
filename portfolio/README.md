# SPEC 重製範例 | Reconstructed SPEC Samples

本資料夾內的規格文件為**作品展示用之重製範例**：文件架構、撰寫方法與顆粒度沿用本人於實際工作中的產出習慣，但所有產品名稱、介面描述、數值、業務規則細節均為虛構，**不含任何前雇主之機密資訊**。完整的實際工作文件可於面談時現場展示。

The documents in this folder are **reconstructed samples for portfolio use**: the structure, method, and level of detail follow my real work output, but all product names, UI descriptions, numbers, and business rules are fictional. **No confidential information from any former employer is included.** Full real-work documents can be shown in person during an interview.

## 撰寫原則 | Writing Principles

- **工程師拿到就能開工**：每個欄位、每個按鈕、每個狀態都有明確定義，不留模糊空間。
- **矩陣化狀態管理**：狀態多的功能一律用「狀態 × 影響面」矩陣表達，避免文字描述漏掉組合。
- **Q&A 收邊界**：規格主體之外，用 Q&A 記錄評審會議中釐清的邊界案例，讓決策有跡可循。
