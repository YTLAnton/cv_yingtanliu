# RWD 響應式設計規格書 (SPEC_DRAFT)

## 1. 專案背景
目前的 `index.html` 採用固定寬度的容器 (`max-w-5xl`) 與 Grid 佈局 (8:4)，在桌面版顯示良好，但在行動裝置或小螢幕上會出現縮放比例過小或需要水平捲動的問題，不利於閱讀。

## 2. 目標 (Goal)
優化 `index.html` 的佈局，使其能根據螢幕寬度自動調整，確保在手機、平板與桌面上皆有最佳的閱讀體驗。

## 3. 人物誌 (Persona)
- **招募經理/面試官**：在通勤或會議間隙使用手機/平板快速瀏覽求職者履歷。
- **一般訪客**：透過社群軟體或行動裝置連結進入履歷頁面。

## 4. 驗收標準 (Acceptance Criteria)
1.  **佈局調整**：
    -   **桌面版 (> 1024px)**：維持現有雙欄設計 (主內容 8 : 側邊欄 4)。
    -   **平板/手機版 (< 1024px)**：改為單欄流式佈局 (主內容在上，側邊欄在下，或視重要性調整順序)。
2.  **內容適配**：
    -   修正 Padding 與 Margin，避免在小螢幕上浪費過多空間。
    -   調整字體大小，確保在行動裝置上清晰可讀。
    -   確保圖片或裝飾性元素 (如 timeline dot) 不會跑版。
3.  **既有功能維持**：
    -   多語系切換功能正常。
    -   列印功能 (`print`) 樣式不受影響或同步優化。

## 5. 限制與考量
-   維持使用 Tailwind CSS (CDN) 與原生 CSS。
-   盡量減少對既有 HTML 結構的大幅重構，優先透過 CSS Class 調整。
